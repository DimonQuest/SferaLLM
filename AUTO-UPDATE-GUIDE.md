## 🔧 Как это работает

### Автоматическая проверка
- При запуске приложения
- Каждые 4 часа в фоновом режиме

### Процесс обновления
1. **Проверка** - приложение проверяет GitHub Releases
2. **Загрузка** - автоматически загружает новую версию
3. **Уведомление** - показывает прогресс и уведомление
4. **Установка** - устанавливается при перезапуске

### UI уведомления
- **Загрузка** - прогресс-бар с процентами
- **Готово** - кнопки "Перезапустить" и "Позже"
- **Позиция** - правый верхний угол
- **Анимация** - плавное появление

## 📦 Настройка для вашего проекта

### 1. Создайте GitHub репозиторий
```bash
# Инициализируйте git (если еще не сделано)
git init
git add .
git commit -m "Initial commit"

# Создайте репозиторий на GitHub
# Затем добавьте remote
git remote add origin https://github.com/ВАШЕ_ИМЯ/sferallm.git
git push -u origin master
```

### 2. Обновите package.json
Замените `yourusername` на ваше имя пользователя GitHub:
```json
"repository": {
  "type": "git",
  "url": "https://github.com/ВАШЕ_ИМЯ/sferallm"
},
"build": {
  "publish": {
    "provider": "github",
    "owner": "ВАШЕ_ИМЯ",
    "repo": "sferallm"
  }
}
```

### 3. Создайте GitHub Token
1. Перейдите на https://github.com/settings/tokens
2. Generate new token (classic)
3. Выберите scope: `repo` (полный доступ)
4. Скопируйте токен

### 4. Настройте переменную окружения
**Windows:**
```bash
setx GH_TOKEN "ваш_токен_здесь"
```

**macOS/Linux:**
```bash
export GH_TOKEN="ваш_токен_здесь"
# Добавьте в ~/.bashrc или ~/.zshrc для постоянного использования
```

### 5. Соберите и опубликуйте
```bash
# Сборка с автоматической публикацией
npm run dist:win -- --publish always

# Или сначала соберите, потом опубликуйте вручную
npm run dist:win
# Затем загрузите файлы на GitHub Releases вручную
```

## 🚀 Публикация обновления

### Способ 1: Автоматическая публикация
```bash
# 1. Обновите версию в package.json
# "version": "2.1.1"

# 2. Создайте git tag
git tag v2.1.1
git push origin v2.1.1

# 3. Соберите и опубликуйте
npm run dist:win -- --publish always
```

### Способ 2: Ручная публикация
```bash
# 1. Соберите
npm run dist:win

# 2. Создайте Release на GitHub
# - Перейдите на https://github.com/ВАШЕ_ИМЯ/sferallm/releases
# - Нажмите "Create a new release"
# - Tag: v2.1.1
# - Title: v2.1.1
# - Описание: changelog

# 3. Загрузите файлы из release/
# - SferaLLM-Setup-2.1.1.exe
# - SferaLLM-2.1.1-portable.exe
# - latest.yml (важно!)
```

## 📋 Формат latest.yml

Файл `latest.yml` создается автоматически при сборке:
```yaml
version: 2.1.1
files:
  - url: SferaLLM-Setup-2.1.1.exe
    sha512: ...
    size: 123456789
path: SferaLLM-Setup-2.1.1.exe
sha512: ...
releaseDate: '2026-04-29T10:00:00.000Z'
```

**Важно:** Всегда загружайте `latest.yml` вместе с установщиком!

## 🧪 Тестирование автообновлений

### Локальное тестирование
1. Соберите версию 2.1.0
2. Установите её
3. Измените версию на 2.1.1
4. Соберите и опубликуйте 2.1.1
5. Запустите установленную 2.1.0
6. Должно появиться уведомление об обновлении

### Проверка вручную
В приложении откройте DevTools (F12) и выполните:
```javascript
// Проверить обновления
window.electronAPI.app.checkForUpdates()
```

### Проверка конфигурации
```bash
# Проверить что GH_TOKEN установлен
echo %GH_TOKEN%  # Windows
echo $GH_TOKEN   # macOS/Linux

# Проверить package.json
cat package.json | grep -A 5 "publish"
```

### Частые проблемы

**Обновление не находится:**
- Проверьте что Release опубликован (не draft)
- Проверьте что latest.yml загружен
- Проверьте версию в package.json

**Ошибка загрузки:**
- Проверьте интернет соединение
- Проверьте что файлы доступны публично
- Проверьте размер файлов

**Не устанавливается:**
- Закройте все экземпляры приложения
- Проверьте права администратора
- Проверьте антивирус

```typescript
autoUpdater.on('update-downloaded', (info) => {
  // Отправить событие в аналитику

});
```

## 🎯 Best Practices

1. **Semantic Versioning** - используйте semver (2.1.0, 2.1.1, 2.2.0)
2. **Changelog** - всегда добавляйте описание изменений
3. **Тестирование** - тестируйте обновление перед публикацией
4. **Rollback** - держите предыдущую версию доступной
5. **Уведомления** - информируйте пользователей о важных изменениях
